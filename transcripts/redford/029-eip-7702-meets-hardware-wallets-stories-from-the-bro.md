And from my talk, if there are two things that I want you to remember, this is the first thing. Myth one, DAPs will drain your account. No, they will not. No wallet wants to support allowing DAPs to trigger authorizations. Authorizations are something that is meant for the wallets to use and DAPs are meant to communicate to the wallets with existing APIs like 5792, 7715. So DAPs talk to wallets through capability APIs, not through authorizations. And then DAP tells the wallet what they request and wallet decides how to deal with this request. So this is a very, very important nuance. To recap, wallets stay in control with EIP7702 and DAPs don't get new powers or cannot move your funds without you confirming. And the second thing that I want you to remember is EIP7702 isn't code storage. This is a myth. Even some hardware wallets still believe that many C-contract delegation is not compatible with code storage. But in reality, what I want you to remember is that all accounts have logic, even EUIs. Just their logic is hidden inside the protocol. And changing that with EIP7702, it simply moved the logic away from the application level towards the delegator, to the smart contract level. And EIP7702, as I said, doesn't break code storage security. It exposes the same logic that EUIs already rely on. It just moves it from the application level.
