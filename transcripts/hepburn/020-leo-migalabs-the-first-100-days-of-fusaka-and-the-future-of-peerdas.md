# Leo (MigaLabs) The first 100 days of Fusaka and the future of PeerDAS

So good morning, everyone. My name is Leo. I'm the founder and the research leader of Micalabs. Thank you for coming. Today, I'm going to talk about the first 100 days of FUSACA and the future of PRDS. So just to do a small recap of what is PRDS. So this is a solution to avoid downloading the whole blob data. So we know that on the roll-up centric scaling roadmap, we have now blobs of data where roll-ups can put everything, all the data they need, in order to have inherited the security of Layer 1. And so we have a number of blobs that are submitted to the L1 at every slot. And the idea is that we want to scale this number of blobs so that the roll-ups can put more and more data as we try to scale, but we don't want the nodes in the Layer 1 to need to download all the data from the blobs. So for that, what we are trying to do is the following thing, which is we extend the data with Reed-Solomon encoding, and instead of downloading the whole data set, you only download part of it. In particular, we download only a certain number of columns. So I believe today any full node in the network is downloading four columns. If you are a super node, that means you are running many, many validators on your node, or for example, you are a block explorer and you want to have all the data, then you are what is called a super node, and then you download all the columns and you download the full data. But the rest of the nodes do not need to download all the data, they only download a number of columns. So the black columns here are the ones that, for example, that you will be downloading. And so that is basically FUSACA, okay? That is, in a nutshell, what FUSACA is, sorry, PRDS is introducing in the FUSACA hard fork. Now, the purpose of this talk is to explain and to show what has happened since the FUSACA hard fork and the BPOs. So the FUSACA hard fork happened in December 3rd of 2025, so last year, a few months ago. Now, if we want to talk about the future of PRDS, there are several improvements that are coming. One of those is cell level dissemination. So this is important because we want to reconstruct data. But today, to reconstruct data, you need at least half of the row, basically. But the only way you will have half of the row is if you have actually half of the columns. And so that means basically the only nodes that can reconstruct in the network are supernodes. So that's bad because then we are relying heavily on supernodes. And you know, that's not great for decentralization. So cell level dissemination is a step forward in that if we can disseminate cells or particular cells on the columns, then we don't need to have the entire dataset in order to recreate the other half or to re-seed the network. So this is an improvement and it's already on progress. I think several of the clients have already implementations working on this. And it's probably going to come in the coming months. We will have cell level dissemination. So this is one of the important things. Now, I will argue that it's important also to move to two-dimensional PRDS. Now, this is a little bit controversial, but not much, I think. But yeah, some people say that we don't need to go there yet, that with one-dimensional PRDS, we are okay. I think we will be okay up to some extent. I think if we keep increasing the number of blobs, disseminating columns and sampling columns instead of sampling cells, it's not really optimal. I think the two-dimensional scheme, which it doesn't need to be so fully two-dimensional, we can actually extend vertically just a number of rows. And just that will already help us to reconstruct data much easier and will make the whole scheme much more robust. And also for sampling, if you are sampling cells instead of sampling columns, you will be reducing the bandwidth on the network. And so that's also another benefit. So I think it's important to keep this in mind. If we can move towards something that is perhaps not full-dust yet, but at least partially two-dimensional, I think it would be good for the network. And finally, more further into the future, we are looking at post-quantum blobs. So what that means is that right now for disseminating the blobs, we rely on KCG commitments in order to make sure that when you receive a cell from any node in the network, you can easily know that this is actually part of the polynomial of the original blob. And the problem is that KCG commitments can be broken with quantum computers. And so we cannot rely on KCG commitments in the future after quantum computers are here. And so the question is, what can we use? And so there is a lot of research already in post-quantum signatures using lattice-based technology, and so on and so forth. But the problem is that when you use those techniques and you recreate the cells, then the proof that you generate is going to be different than the original proof that was disseminated. And this is a problem because then you cannot really re-seed these reconstructed cells into the network. So basically, at this point, we have really no solution on how to succeed to have blobs in a post-quantum world. But, you know, the good news is that there is a team that is very heavily working on it, and we hope that by the time quantum computers reach us, we will have already a working solution. So yeah, that was my talk on the future of PRDS. If you have questions, I'll be happy to answer. Thank you.
