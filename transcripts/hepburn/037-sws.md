I'm going to present research that we did jointly with Babook, who is here, about how the block building process affects Ethereum UX, essentially. Now, I think we all more or less know about this strange way in which Ethereum transaction originates from a user or a wallet and ends up eventually included on-chain. There is typically a user sending its transaction either through the public mempool, but now more and more through some things called private RPC endpoints, order flow originators, private mempools. These are systems that relay transaction privately to builders without going through the public mempool. So those transactions reach builders, they package into blocks, they then send those blocks to relays. Relays then run a sort of auction, giving, there are multiple relays, so it's like an auction with multiple auctioneers. And then the proposer will ask the relays to give them the highest paying block. They will then give the block to the proposer and finally the proposer is going to append this block to the chain. Now, now that the protocol proper, so what do we mean by Ethereum, really starts with the proposer with the block, but of course everything in between matters from the user point of view because it determines many things about like his own experience of using the chain.
