# Luis Correia (Flashbots) Optimising Block Building with Genetic Algorithms

Okay, so today I'm going to be talking about how we can optimize block building using genetic algorithms. All right, so let's start off with the basics. What even is block building? So builders are these specialized entities that construct blocks. And what this basically means is they try to pack transactions in a way that maximizes the block value. And these blocks that they construct, they then send them over to proposers, who have a very simple job of just proposing that block and broadcasting it to the network. And so we have this separation, right? The block building computation is done by the builders, and then validators have this simple job. And so we call this proposer-builder separation. And it was added to add more decentralization to the validator set. Importantly, block building is a competitive auction where bidders try to outbid each other. And so whoever manages to bid the most gets their block proposed. And so before we go any further, this concept of MEV is actually important. I'm sure many of you have heard of it. And it's the additional value that a block proposer or a builder or a sequencer can extract by controlling which transactions get included in the block and how they're ordered. And there's different forms of MEV, some that can be harmful, like sandwich attacks. But then there's also beneficial forms, like arbitrage and liquidations. And so during block building, we aim to maximize the beneficial kind. So following on from this, what makes a block valuable? Well, first of all, every transaction pays priority fees. So if we have a block where the transactions are paying higher in priority fees than another block where the priority fees are lower, then that block is going to be more valuable. But we have these searchers who are separate entities that scan public and private mempools and scan for MEV opportunities. And they craft bundles to extract this MEV. And then they submit these bundles over to the builders and pass on some share of that MEV that they extracted over to the builder. And so you can see in this diagram, we've got a searcher that finds some transaction they want to backrun. They then form a bundle with that user transaction plus their backrun transaction and then submit it over to the builder. And so you can think of block value as the sum of the priority fees plus this additional MEV. And block builders are going to be trying to maximize this by selecting and ordering transactions and bundles in a certain way. All right, so this is kind of like a very high level overview of block building. So the first step is this intake of order flow.
