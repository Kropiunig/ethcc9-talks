So, XMSS, or Extended Merkle Signature Scheme, is a post-quantum signature scheme built entirely from hash functions, so its security does not rely on elliptic curves or other algebra that quantum computers can break easily. At the leaf level, XMSS uses Winternets one-time signatures. Each leaf can only be used one time for signing something securely, because if you signed multiple messages with that signature, a quantum computer could use that, like the information, to forge a new signature, which would break the security. XMSS places basically, like, many Winternets one-time signatures in each leaf. So you could think of it as like a Merkle tree, where each leaf is a key that we can use once to sign a block or a slot, and then at the end, you could say their identity is the root of the Merkle tree, which they broadcast to the network. And yeah, this scheme is stateful because the validator needs to know, like, which keys they have used in the past.
